Users
 -UserId: PK,						int
 -Username, 						varchar(100), NN
 -Password: Encrypted, 					varchar(255), NN
 -Code,							varchar(10)
 -Role: FK to RoleID in Roles table, 			int, NN
 -FolderLocation: 					varchar(255)

Roles
 -RoleId: PK, 						int
 -RoleName: 						varchar(50), NN

Subjects
 -SubjectCode: PK,					varchar(50)
 -SubjectName: PK (Not necessary),			varchar(150)
 -FolderLocation (Not necessary):			varchar(255)

Tests:
 -TestId: PK,						int
 -TestName,						varchar(100)
 -Subject: FK to SubjectCode in Subjects table,		varchar(50)


Assignments:
 -UserId: FK to UserId in Users table,			int
 -TestId: FK to TestId in Tests table,			int


Root location + SubjectName + TestName identifies the exact location of the description 
file (desc_TestName name format inside the location) and the test files 
(testXX_TestName name format inside location)

Role Elements:
 -Student:
 can read tests, read/write/delete permission in own folder
 -Teacher:
 can read/create tests, read/write/delete permission in own folder
 -Administrator:
 read/write/delete permission from root directory, can create useraccounts
